프로젝트 제안: 나만의 'To-Do 리스트 우선순위' 예측 모델

#### 1. 해결하고자 하는 문제 (Problem)

- 해야 할 일(To-Do)이 여러 개 쌓였을 때, 무엇을 먼저 처리해야 할지 결정하는 것이 혼란스럽고 시간이 걸립니다. 해야 할 일이 많은 상황에서, 어떠한 일을 먼저 처리해야 할 지 고민하는 과정에 생각보다 많은 시간 소요되었던 경험이 있습니다. 또한 주관적인 감정에 휘둘려 중요한 일을 미루고, 쉽거나 급해 보이는 일을 먼저 처리하는 비효율적인 의사결정을 반복하게 됨을 느꼈습니다.

#### 2. 실제 사용 시나리오 (Personal Use Case)

- 매일 아침(또는 새로운 과제/업무가 4~5개 이상 쌓였을 때), 이 도구(스크립트)를 실행합니다. 그 후 오늘 해야 할 일 목록과 각 항목의 정보(예상 시간, 마감 기한, 일의 종류 등)를 입력합니다. 모델은 각 할 일의 '예측 우선순위 점수'(예: 9.5점) 또는 '등급'(예: 'A-긴급/중요')을  반환할 것이고, 이 객관적인 점수를 참고하여 우선순위가 높은 일부터 순서대로 업무를 효율적으로 처리할 수 있을 것 입니다.

#### 3. 제안하는 솔루션 (Solution)

- '나의 과거 의사결정 패턴'을 학습한 개인 맞춤형 ML 모델을 구축합니다. '할 일 텍스트', '마감 기한', '예상 소요 시간', '업무 유형' 등 여러 요소를 입력받아, 이들을 feature로 이용해 '내가 생각하는 실제 우선순위'(Label)를 예측하는 **회귀(Regression)** 또는 **분류(Classification)** 모델을 만들 것 입니다. 간단한 터미널(CLI) 스크립트나 Streamlit 웹 앱을 통해 이 모델을 쉽게 사용할 수 있도록 구현할 생각입니다.
#### 4. 주요 기능

- 새로운 할 일 등록 (텍스트, 예상 시간, 마감일, 업무 유형 등 입력)
- 등록된 할 일 목록의 '우선순위' 일괄 예측 및 정렬
- 또한 오늘의 실제 우선순위(정답)를 피드백하여 학습 데이터를 쉽게 축적할 수 있는 기능도 구현하고자 합니다.

#### 5. ML 활용 계획

- **Assignment 4: (데이터 수집 및 라벨링)**

    - **데이터 수집 단계**: 향후 2~3주간 직접 '할 일' 데이터를 수집합니다.
    - **Features (X):** '할 일 내용'(Text), '예상 소요 시간(분)'(Num), '마감 기한(D-day)'(Num), '업무 유형'(Cat, 예: 과제/공부/개인)
    - **Label (Y):** '내가 실제로 판단한 우선순위 점수(1-10점)' (=Regression) 또는 '등급(High/Mid/Low)' (=Classification)
    - 최소 50~100개의 (X, Y) 데이터 쌍을 확보하는 것을 목표로 합니다.
        
- **Assignment 5: (모델 학습 및 평가)**
	
	**목표**: 모든 특징 벡터를 **하나로 연결하여 최종 입력 데이터셋 구축**하여 매일 새로운 TO-DO-LIST가 주어지면 우선순위 매기기
	**평가:** 데이터 수가 적으므로 **K-Fold Cross-validation**을 이용하고자 합니다.
	
- **Assignment 6: (실제 사용 및 개선)**
    
    - Assignment 5에서 학습된 최적의 MLP 모델을 실제 스크립트에 탑재합니다.
    - 매일 새로운 할 일을 입력하며 모델의 예측값을 실제로 사용합니다.
    - 이용할 때 마다 실제 우선순(정답)을 매겨 추가적인 학습 데이터를 쌓습니다. 모델이 탑재된 스크립트를 이용하며 피드백을 주면 줄 수록, 학습 데이터 양이 늘어나기 때문에, 재학습을 통해 정확도가 더 오를 것으로 기대합니다.
